# Мобильное приложение (APK) через Render + PWABuilder

Принцип тот же, что у PharmaPrice, но с одним добавленным звеном: движок (yt-dlp)
живёт не на ПК, а на бесплатном хостинге **Render**, который раздаёт PWA по HTTPS.
Этот HTTPS-адрес прогоняется через **PWABuilder** → получается APK/AAB.
Приложение работает **без компьютера**, файлы качаются прямо в телефон.

Содержимое этой папки `cloud/` — и есть то, что нужно выложить на Render.

---

## Шаг 1. Залить код на GitHub
1. Создай аккаунт на https://github.com (если нет) и новый **пустой репозиторий**.
2. Загрузи в него **все файлы из этой папки `cloud`** (server.py, index.html,
   Dockerfile, requirements.txt, render.yaml, manifest.webmanifest, sw.js, иконки).
   Можно прямо через кнопку **Add file → Upload files** на сайте GitHub.

## Шаг 2. (для YouTube) добавить cookies.txt
YouTube блокирует анонимные сервера. Чтобы он качался:
1. Возьми файл `cookies.txt` (как делали на ПК — расширение «Get cookies.txt LOCALLY»,
   на youtube.com → Export).
2. Загрузи его в тот же репозиторий рядом с `server.py`.
> Pinterest, TikTok, VK и большинство сайтов работают и без cookies.
> Cookies со временем устаревают — иногда нужно обновлять файл в репозитории.

## Шаг 3. Поднять сервер на Render
1. Зайди на https://render.com, войди через GitHub.
2. **New → Web Service** → выбери свой репозиторий.
3. Render сам увидит `Dockerfile`. Проверь: **Plan = Free**, регион любой.
4. Нажми **Create Web Service** и подожди сборку (несколько минут).
5. Получишь адрес вида **https://video-downloader-xxxx.onrender.com** — открой его,
   убедись, что интерфейс грузится и что-нибудь скачивается.

## Шаг 4. Проверить на телефоне
Открой этот адрес в Chrome на телефоне, вставь ссылку, нажми «Скачать» —
файл должен сохраниться в загрузки телефона.

## Шаг 5. Сделать APK через PWABuilder
1. Зайди на https://www.pwabuilder.com
2. Вставь свой Render-адрес → **Start**.
3. **Package For Stores → Android → Generate Package**.
4. Скачается zip: внутри `*.apk`, `*.aab`, `signing.keystore`, `assetlinks.json`
   (ровно как было у PharmaPrice). Ключ и пароль **сохрани** — пригодятся для обновлений.

## Шаг 6. Убрать адресную строку (Digital Asset Links)
Чтобы приложение открывалось как родное (без полоски браузера):
1. Возьми `assetlinks.json` из скачанного пакета PWABuilder.
2. Загрузи его в репозиторий рядом с `server.py` (имя — `assetlinks.json`).
3. Render сам передеплоит. Сервер отдаёт его по адресу
   `https://<твой>.onrender.com/.well-known/assetlinks.json` — это и нужно TWA.

## Шаг 7. Установить APK
Перекинь `.apk` на телефон, открой, разреши «установку из этого источника» — готово.
Иконка появится на главном экране, приложение работает само по себе.

## Честные ограничения бесплатного Render
- После простоя сервис «засыпает» — первая загрузка после паузы идёт ~30–60 сек.
- Слабый CPU и мало места: очень большие/длинные видео могут долго идти или обрываться
  (файлы на сервере временные и сами удаляются).
- YouTube периодически блокирует IP сервера — лечится свежим `cookies.txt` (Шаг 2).
- Качай только то, на что у тебя есть права. Публичный сервер-качалка может нарушать
  правила хостинга — это вариант для личного использования.

## Обновить приложение позже
Меняешь файлы в репозитории (например, новый `cookies.txt`) → Render передеплоит сам.
Если менял интерфейс/манифест — можно заново прогнать через PWABuilder тем же ключом
(`signing.keystore`), чтобы обновить APK.
